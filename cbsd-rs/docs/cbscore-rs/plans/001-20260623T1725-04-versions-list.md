## Goal

`cbsbuild versions list --from <s3-host-url>` lists the releases in
`config.storage.s3.releases`, printing each release id (and, under `-v`,
per-arch build type / OS version and per-component version/sha1/repo-url/loc).
It **works** — the Python `versions_list` calls `list_releases(secrets, url)`
with two args where four are required (`cmds/versions.py:129`), raising
`TypeError` and never running; the port supplies `bucket`/`loc` from config and
`--from` as the host URL. With no `storage.s3` it fails with a clear,
operator-actionable error, never a panic.

## Commit 1: `versions list --from` — S3 listing + fixed wiring

Owns: 005 (`s3_list`, `list_releases`), 006 (the C7 fix), 010 (CLI). Reuses
`Config` (M2/C2), and `SecretsMgr`/`s3_creds`/`s3_download_str_obj` (M2/C4a —
the download primitive landed there for `release_desc_upload`'s read-modify-
write; review S-7) — only the **listing** operation (`s3_list`) is new here.

- **`s3_list`** (`utils::s3`, 005) — `list_objects_v2` with **continuation-token
  pagination** and `Delimiter = "/"` when `prefix_as_directory`, collecting
  `CommonPrefixes` into `S3ListResult.common_prefixes`; `S3ObjectEntry`
  (`.name()` = basename). This is the one S3 operation the build milestone did
  not need (its reuse path downloads specific keys; it never lists).
- **`list_releases`** (`releases::s3`, 005) — `s3_list` the `<loc>/` prefix as a
  directory, download+parse each `.json`, **leniently skipping** malformed/old
  descriptors with a warning (not fatal); returns `map<version, ReleaseDesc>`.
  The body is correct in Python; only its call site was broken.
- **CLI** (`cbsbuild cmds::versions`, 010) — the `versions list` subcommand
  (`-v/--verbose` flag; **required** `--from ADDRESS`), loading the config for
  Vault/secrets. **The fix**: `bucket`/`loc` come from
  `config.storage.s3.releases`, `--from` supplies the host URL; both
  `config.storage` and `config.storage.s3` are optional, so an absent S3 config
  yields a clear "no S3 storage configured; `versions list` requires
  `storage.s3`" error. Output is **approximate human-readable text**, not
  byte-for-byte parity (no machine consumer) — release id, and under `-v` the
  per-arch and per-component detail.

**Testable:** `versions list --from <url>` lists releases present under
`storage.s3.releases`; a listing spanning multiple pages returns all objects;
`list_releases` skips a malformed descriptor and returns the valid ones; a
config without `storage.s3` errors clearly (no panic, no `TypeError`); `-v` adds
the per-arch/per-component detail.
