# Phase M1 — Versions create (C1)

Derived from the capability commit map in `design/001` (M1/C1). Delivers the
first operator-facing command, `cbsbuild versions create [VERSION]`. It is
**config-independent** (CLI flags + the local git repo only), so no config types
land here. Owning designs: 006 (versions), 003 (subprocess/redaction/git), 010
(CLI surface), 002 (wire types — landed with this first consumer).

## Goal

`cbsbuild versions create [VERSION]` writes a `VersionDescriptor` JSON to
`<git-root>/_versions/<type>/<version>.json` whose field names and nesting match
Python's `model_dump_json` output (plus the new `schema_version` marker). An
omitted `VERSION` auto-generates a sortable **UUIDv7**; a supplied version must
be at least `M.m.p`. Secrets that ever flow through a command line are
structurally un-loggable (`SecureArg`). The trailing "image descriptor missing?"
note fires correctly (fixing the broken Python `get_image_desc`) and is skipped
for UUIDv7 versions.

## Commit 1: version wire types + parse/validate helpers (pure)

Pure, zero-IO logic — the type and parse layer the CLI builds on. Mirrors the
repo's established "foundation" commit precedent
(`cbsd-rs/docs/cbsd-rs/plans/002-…-01-foundation.md` Commit 1: pure types
validated by round-trip tests). Owns: 002 (types), 006 (parse/type table).

**`cbscore-types`:**

- `version.rs` — `VersionDescriptor` + `VersionSignedOffBy`, `VersionImage`,
  `VersionComponent` (`#[serde(rename = "ref")]`), snake_case, no `rename_all`
  (002).
- `schema.rs` — the `schema_version` machinery: `schema_v1()` default, the
  `UnknownSchemaVersion { format, found, max }` post-deserialize validation step
  (002 — a higher marker is a hard error, **not** a serde attribute).
- `version_type.rs` — `VersionType { Release, Dev, Test, Ci }`
  (`rename_all = "lowercase"`).
- `error.rs` — type-layer taxonomy: `MalformedVersion`, `VersionError`,
  `InvalidVersionDescriptor { path }`, `UnknownSchemaVersion` (thiserror, pure).
- `store.rs` — the pure
  `descriptor_path(root, type, version) -> <root>/<type>/<version>.json` helper
  (006 — "the one source of truth for the path"). **Landed now, not deferred to
  M5 (review S-6):** M1's writer uses it with the hardcoded `_versions` root;
  M5/C9 adds only the configurable root + `--versions-dir` flag on top, so the
  path-construction logic is never duplicated (protects on-disk-layout parity,
  invariant 3).

**`cbscore` (no IO yet — these need `regex`, so they live in the library, not
`cbscore-types`):**

- `versions/parse.rs` — `parse_version`, `get_major_version`,
  `get_minor_version`, `normalize_version`, `parse_component_refs`
  (`COMPONENT@REF`), the single grammar regex (006 §"Version strings").
- `versions/version_type.rs` — the `label → (VersionType, description)` table
  with `get_version_type` (name→type **lookup**, not a regex parse — resolves
  000's M3) and `get_version_type_desc`.
- `versions/validate.rs` — `validate_version` (accepts a `M.m.p` Python shape
  **or** a valid UUIDv7) and `resolve_version(Option<&str>)` (supplied → use;
  omitted → fresh UUIDv7).

**Testable:** port the Python parametric parse/normalize cases
(`versions/utils.py:161-241`) **verbatim** as golden tests; `get_version_type`
maps each label and rejects unknown with `VersionError` (and provably does not
parse version strings); `validate_version` accepts `20.2.1` and a UUIDv7 but
rejects `19`, `19.2`, `foobar`; `VersionDescriptor` round-trips
(serialize→parse→equal) and a marker-less blob parses as v1 while a higher
marker yields `UnknownSchemaVersion`; a golden-file test asserts the JSON shape
matches Python's `model_dump_json` (compared as parsed values, marker
accounted); `descriptor_path(root, type, version)` yields
`<root>/<type>/<version>.json`.

## Commit 2: subprocess + redaction + git + `versions create` CLI

Lands the lowest IO layer (subprocess primitive + redaction + the two C1 git
ops) **together with its first real consumer** (`version_create_helper` reads
the git user/repo-root; the CLI runs the command), so nothing here is dead code.
Owns: 003 (primitive/redaction/git), 006 (create/title/UUIDv7/image-desc), 010
(CLI parsing + globals).

**`cbscore` subprocess + redaction layer** (`utils::subprocess`, `utils::git` —
lift-out candidates, own tracing targets, deps only `tokio`/`tracing`/
`thiserror`/`regex`/`camino`/`which`):

- `utils/subprocess.rs` —
  `run_cmd(&[CmdArg], RunOpts) -> Result<CmdOutput, CommandError>`: inherit
  env + `extra_env`; piped stdout/stderr read concurrently line-by-line through
  an optional async `out_cb`; `tokio::time::timeout` wrapping spawn+wait,
  killing+reaping the child on elapse/cancel; **timeout is a distinct
  `CommandError` variant** (separated from non-zero exit, resolving the Python
  FIXME); non-zero exit returns `CmdOutput`, spawn failure is a typed
  `CommandError`. Plaintext rendering used **only** at spawn.
- `utils/redact.rs` — `trait SecureArg` **without** `Debug`/`Display`
  supertraits (compiler-enforced redaction — invariant 4);
  `enum CmdArg { Plain(String), Secure(Arc<dyn SecureArg>) }` with hand-written
  `Debug`/`Display` emitting `redacted()`; `Password`, `PasswordArg`,
  `SecureUrl`; the plain-string pattern backstop (`--pass`/`--passphrase`/
  `--password`/`-p`, two-token and inline — strictly broader than Python).
- `utils/git.rs` — `run_git(args, path: Option<&Utf8Path>)` (prepends `git`,
  `-C <path>` when given); `GitError { retcode, msg }` + `GitConfigNotSet`; the
  two C1 ops only: `get_git_user()` and `get_git_repo_root()` (clone/checkout/
  worktree/apply land at M2/C3).

**`cbscore` versions create + image descriptor:**

- `versions/create.rs` — `version_create_helper(...) -> create(...)` assembling
  a `VersionDescriptor` from CLI inputs; `_do_version_title` (type-description +
  `M.m.p` + suffix-part uppercasing, e.g. `rc.1` → `(RC 1)`); the UUIDv7-version
  title `… version created at <ISO-8601 from the UUIDv7-embedded timestamp>`;
  `image.tag` falls back to the version string; the `COMPONENT=URI` override
  parsing (006 owns it; `load_components`/`CoreComponent` is 007's, referenced
  but **not** built here — `versions create` only needs the repo URL per ref,
  resolved from component defs/overrides). Write site via
  `descriptor_path(<git-root>/_versions, type, version)` (the shared helper from
  Commit 1 — **not** an inline path, review S-6); existing path → "version
  already exists" error. The root is the hardcoded `<git-root>/_versions`
  default; M5/C9 adds only the configurable root + `--versions-dir`, reusing the
  same helper.
- `versions/image_desc.rs` — `ImageDescriptor`/`ImageLocations` types (002
  deferred to 006; external, **unversioned**) and the **fixed** `get_image_desc`
  (implements the intended contract, not the broken raw-string Python filter):
  extract `M.m.p` via `parse_version` (leading `v` optional); cheap substring
  pre-filter over `<git-root>/desc/*.json`; authoritative match = exact version
  membership in a descriptor's `releases` (un-normalized form); skip malformed
  candidates; two matches → `ImageDescriptorError`, none → `NoSuchVersion`.
  IO-triggered errors (`NoSuchVersion`, `ImageDescriptorError`,
  `NoSuchVersionDescriptor { path }`) live here (002 §error split).

**`cbsbuild` CLI** (`cbsbuild`):

- `cli.rs` — the clap root with globals `-c/--config <path>` (defaulting to
  `cbs-build.config.yaml`, **not** `required`; not validated at parse time) and
  `-d/--debug`; the `versions create [VERSION]` subcommand with the exact flag
  table from 010 §"versions create" (`-t/--type` default `dev`; **required
  repeatable** `-c/--component NAME@VERSION`; repeatable `--components-path`,
  `-o/--override-component-uri`; `--distro` default `rockylinux:9`;
  `--el-version` default 9; `--registry`, `--image-name` defaults; optional
  `--image-tag`; `VERSION` **optional** → UUIDv7).
- `bool_parser.rs` — the shared Click-equivalent `BOOL` value parser
  (`1/true/t/yes/y/on` ⇒ on; counterparts/empty/unset ⇒ off) used by `CBS_DEBUG`
  (and later `--tls-verify`); **not** a clap presence flag — `CBS_DEBUG=0` is
  off (resolves H3). Tracing-subscriber setup keyed off the resolved debug level
  (targets from `cbscore-types`).
- `cmds/versions.rs` — the `versions create` handler: build inputs, call
  `version_create_helper`, write the descriptor, then run the (non-fatal)
  `get_image_desc` note — **skipped** for a UUIDv7 or patch-less version.
  `versions create` never loads the config (010).

**Testable:** `cbsbuild versions create 20.2.1 -c ceph@v20.2.1` writes a
descriptor whose on-disk layout matches Python; omitting `VERSION` yields a
valid UUIDv7 `desc.version` and a "created at <ts>" title and **skips** the
image-descriptor check (no error); structural redaction tests —
`format!("{:?}", cmd)` and the primitive's logged form contain `<CENSORED>`,
never the plaintext, for `Password`/`PasswordArg`/`SecureUrl`, and a stub proves
plaintext reaches `exec` while the log shows the redacted form; `CBS_DEBUG=0` →
debug off; `get_image_desc` resolves a version present in a descriptor's
`releases`, and a missing `desc/` dir or no match → `NoSuchVersion`, two matches
→ conflict.

## As-built (C1 complete)

Landed as `5d6d890` (types/parse), `6ef1d27` (2a, write descriptor), and
`f81cd86` (2b, image-descriptor note). Where the realisation refined this plan:

- **Commit 2 split into 2a + 2b** (agreed at implementation time). With a
  component loader added (below), the single commit reached ~1100–1260 lines, so
  it split along the one clean seam: **2a** writes the descriptor (subprocess +
  redaction + git + component loader + create/title + CLI — the tightly-coupled
  chain S-5 says cannot split further without orphaning git); **2b** adds the
  non-fatal `get_image_desc` "missing" note. Each kept the subprocess/git layer
  with its real CLI consumer.
- **A minimal component-definition loader was needed in M1**, not deferred
  to 007. `create` rejects any `-c X@ref` whose component `X` is not a loaded
  definition (an override does not bypass that), so `versions create` cannot
  build a descriptor without loading defs. `cbscore::components` reads only
  `name`+`repo` from `cbs.component.yaml` (ignoring the `build`/`containers`
  sections, which the full `CoreComponent` model adds in 007/C3). **Parity
  caveat (impl-review F2):** because it ignores those sections, the M1 loader
  accepts a `cbs.component.yaml` that omits `build`/`containers`, which Python's
  `CoreComponent.model_validate` rejects. This leniency is intentional for M1
  (only `repo` is needed), but **007/C3 must restore strict validation** when it
  lands the full model, so a genuinely malformed component file is caught.
- **Deferred to their first consumers** (lands-with-consumer): the async
  `out_cb` (runner streaming, M2), `PasswordArg` and `SecureUrl` (C3 git clone /
  C4–C5 flags). Only `Password` + the credential-flag backstop land now, which
  fully establish and test invariant 4.
- **The version-store subdir uses the canonical `VersionType::as_str()`** (lower
  case), not Python's raw `-t` string — identical for the lowercase type names
  that validate, and it removes a latent Python inconsistency (design 006 keys
  the store on the canonical type).
- **`cbsbuild` uses explicit `ExitCode` per error** (010's 0/1/2 scheme) rather
  than an anyhow-at-`main` collapse; `anyhow` was dropped from `cbsbuild`.
- **`--config` is a non-global root arg** so `-c` is free for
  `versions create -c/--component` (clap parses root options before the
  subcommand, matching Click's group-option ordering).
