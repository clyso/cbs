# 010 — CLI surface (`cbsbuild`)

This is the reference design for the `cbsbuild` command-line surface: the clap
command tree, the per-subcommand flags (the complete parity table that resolves
000's M1), the global `--config`/`--debug` handling, the dispatch into the
runner and the version/release helpers, and the deliberate command-level drops.
It is the single source of truth for **what the binary parses** and **how each
command maps to a subsystem**; the subsystems themselves are owned elsewhere
(009 runner, 006 versions, 007 builder, 004 config/secrets).

Source of truth: `cbscore/__main__.py` (the root group), `cbscore/cmds/*.py`
(`builds.py`, `versions.py`, `config.py`, `advanced.py`, `__init__.py`), and the
entrypoint the binary replaces (`cbscore/_tools/cbscore-entrypoint.sh`, for the
`CBS_DEBUG` and `HOME` handling).

`cbsbuild` is the same binary on both sides of the two-phase boundary (001/009):
the operator runs `cbsbuild build`/`versions …` on the host, and the runner
mounts the same binary into the builder container where it re-enters as
`cbsbuild runner build` (009). This document owns both shapes and guarantees the
host emit (009) and the in-container parse agree.

## Command tree

```
cbsbuild [--config <path>] [-d|--debug]
├── build <DESCRIPTOR>            host build orchestration → runner::run (009)
├── runner build                  hidden; in-container half → Builder.run (007)
└── versions
    ├── create [VERSION]          write a version descriptor    → 006
    └── list                      list releases from S3         → 006/005
```

Dropped relative to Python (see Dropped surface): the `config` group
(`init`/`init-vault`) and the empty `advanced` group.

## Global options (`--config`, `--debug`)

Both live on the root group, as in Python (`__main__.py:37-56`).

- **`-c/--config <path>`** — Python marks it `required=True` **and** gives it a
  `default` of `cbs-build.config.yaml` (`__main__.py:54-55`). That pairing is a
  Click wart: a default always satisfies "required", so the flag is effectively
  optional-with-default. The port models it as a **global option defaulting to
  `cbs-build.config.yaml`, not `required`** (clap rejects `required` + a
  default, and it would be meaningless anyway). The path is **not** validated at
  parse time (Python uses `exists=False`); only commands that actually load it
  (`build`, `runner build`, `versions list`) read the file and fail if it is
  missing. **`versions create` never loads the config** (its Python handler
  takes no context — `versions.py:266`), so that command works with no config
  file present even though the global default names one.
- **`-d/--debug`** — enables debug logging. Bound to the `CBS_DEBUG` env var
  (H3). See the next section — the env binding is **not** a naive presence
  check.

### `CBS_DEBUG` parsing (resolves H3, precisely)

H3 requires the in-container binary to read `CBS_DEBUG` from the environment
(009 forwards `-e CBS_DEBUG=<1|0>`; there is no `--debug`-flag translation
anymore — the entrypoint's `CBS_DEBUG=1 → --debug` shim is gone). The trap: a
presence-based env binding (clap `ArgAction::SetTrue` with `.env("CBS_DEBUG")`,
or similar) would treat the **non-empty** string `CBS_DEBUG=0` as "set" and turn
debug **on**, silently inverting the off case that 009 explicitly emits.

The port therefore specifies the semantics explicitly rather than relying on a
framework default:

> Debug is **on** iff the `-d/--debug` flag is passed **or** the `CBS_DEBUG`
> environment variable holds a Click-`BOOL`-truthy value. Click's `BOOL`
> coercion is the authority: `1`/`true`/`t`/`yes`/`y`/`on` (case-insensitive)
> are on; `0`/`false`/`f`/`no`/`n`/`off`, empty, and unset are off. **In
> particular `CBS_DEBUG=0` is off.**

This reproduces Click's env-`BOOL` coercion on the host (`__main__.py:39`); it
is a **superset** of the entrypoint's stricter `CBS_DEBUG == "1"` test in the
container (`cbscore-entrypoint.sh:54-55`), which is harmless because the only
emitter (009) sends `1|0`. The implementation must use this value parser (or an
explicit `env::var` read), **not** a presence flag (clap `SetTrue` + `.env`,
which would read non-empty `CBS_DEBUG=0` as on); a test pins `CBS_DEBUG=0` →
off.

## Per-subcommand flags (resolves M1)

The tables below are the complete flag parity table 000's M1 asked for. Columns:
**Shape** — `flag` (bare boolean, no value), `value` (takes one value),
`value×N` (repeatable, collects a list), or `arg` (positional). **Binding** —
`required`, or `default=…`, or `optional`. **Port** — kept as-is, or the
divergence.

### `build <DESCRIPTOR>` (`builds.py:36-131`)

| Option                  | Shape | Binding             | Port                                                                 |
| ----------------------- | ----- | ------------------- | -------------------------------------------------------------------- |
| `DESCRIPTOR`            | arg   | required (existing) | kept                                                                 |
| `--cbscore-path`        | value | **required**        | **dropped** (binary mount, B1/M1)                                    |
| `-e`/`--cbs-entrypoint` | value | optional            | **dropped** (binary mount, B1/M1)                                    |
| `--timeout`             | value | `default=14400.0`   | kept (seconds; 4 h)                                                  |
| `--sign-with-gpg-id`    | value | optional            | kept — overrides `config.signing.gpg`                                |
| `--sign-with-transit`   | value | optional            | kept — overrides `config.signing.transit`                            |
| `--log-file`            | value | optional            | kept — host log sink (009 `RunOpts` log path)                        |
| `--skip-build`          | flag  | `default=false`     | kept                                                                 |
| `--force`               | flag  | `default=false`     | kept                                                                 |
| `--tls-verify`          | value | `default=true`      | kept — **value-taking BOOL**, not a bare flag (`--tls-verify=false`) |

`--tls-verify` is the easy-to-flatten one: Python declares it `default=True`
with no `is_flag` and no `type`, so Click infers `BOOL` and it is a
**value-taking** option, parsed as `--tls-verify=<bool>`. 009 already emits it
in value form into the container; the host parse must match (a bare-flag
modelling would reject `--tls-verify=false`). It must also use the **same
Click-equivalent `BOOL` value parser** as `CBS_DEBUG` (above) — accepting
`1`/`true`/`t`/`yes`/`y`/`on` and their false counterparts, case-insensitive —
**not** clap's default bool parser, which accepts only `true`/`false` and would
reject operator inputs like `--tls-verify=yes`/`on`/`0` that Click accepts. A
single shared `BOOL` value parser covers `--tls-verify` and `CBS_DEBUG`.

### `runner build` (hidden, `builds.py:228-259`)

| Option         | Shape | Binding             | Port                               |
| -------------- | ----- | ------------------- | ---------------------------------- |
| `--desc`       | value | required (existing) | kept                               |
| `--skip-build` | flag  | `default=false`     | kept                               |
| `--force`      | flag  | `default=false`     | kept                               |
| `--tls-verify` | value | `default=true`      | kept (value-taking BOOL, as above) |

This is the in-container contract 009 emits against:
`--config … runner build --desc … --tls-verify=… [--skip-build] [--force]`. The
group is `hidden` (operators use `build`); the port keeps it hidden.

### `versions create [VERSION]` (`versions.py:179-265`)

| Option                          | Shape   | Binding                    | Port                                                 |
| ------------------------------- | ------- | -------------------------- | ---------------------------------------------------- |
| `VERSION`                       | arg     | required → **optional**    | **divergence**: optional; absent ⇒ auto UUIDv7 (006) |
| `-t`/`--type`                   | value   | `default=dev`              | kept                                                 |
| `-c`/`--component NAME@VERSION` | value×N | **required**, repeatable   | kept                                                 |
| `--components-path`             | value×N | optional, repeatable       | kept                                                 |
| `-o`/`--override-component-uri` | value×N | optional, repeatable       | kept                                                 |
| `--distro`                      | value   | `default=rockylinux:9`     | kept                                                 |
| `--el-version`                  | value   | `default=9` (int)          | kept                                                 |
| `--registry`                    | value   | `default=harbor.clyso.com` | kept                                                 |
| `--image-name`                  | value   | `default=ces/ceph/ceph`    | kept                                                 |
| `--image-tag`                   | value   | optional                   | kept                                                 |

`VERSION` optional is a **deliberate divergence** (settled decision; not an
accidental parity gap): when omitted, `version_create_helper` generates a UUIDv7
and derives the title from its embedded timestamp (006). The three repeatable
options (`-c`, `--components-path`, `-o`) and the **required** repeatable
`-c/--component` are the easy-to-flatten shapes — encoded as `value×N` above.

### `versions list` (`versions.py:294-310`)

| Option           | Shape | Binding         | Port                           |
| ---------------- | ----- | --------------- | ------------------------------ |
| `-v`/`--verbose` | flag  | `default=false` | kept                           |
| `--from ADDRESS` | value | required        | kept — S3 address to list from |

Loads the config (`@with_config`) for Vault/secrets, then lists releases (006;
the broken `list_releases` arity and the no-S3-config error are fixed there as
C7). **Output is approximate human-readable text, not byte-for-byte parity** —
there is no machine consumer of `versions list` stdout, so the port reproduces
the _information_ (release id; under `-v`, per-arch build type, OS version, and
per-component version/sha1/repo-url/loc) without pinning Python's exact spacing.

## The two runner consumers

`build` and `runner build` are the host and in-container halves; both ultimately
drive 009/007. 010 owns their parsing and the thin glue around the call.

### `build` — host orchestration glue (thin; fixes a dead-secrets write)

Python's `cmd_build` (`builds.py:133-220`) does five things: load config; fold
`--sign-with-gpg-id`/`--sign-with-transit` into `config.signing` (constructing a
`SigningConfig` if absent); reject a pre-existing `--log-file`; **marshal
secrets to a temp file**; then call `runner(...)`. The port keeps the first
three and the runner call, and **drops the fourth** — it is dead code:

> **Broken Python the port fixes (dead plaintext-secrets write).** `cmd_build`
> creates a temp secrets file and writes the plaintext secrets into it
> (`builds.py:179-186`), but the `runner(...)` call never receives that path
> (`builds.py:191-203`) — `runner` marshals its **own** secrets temp file
> internally (`runner.py:214-222`, 009 step 3) and mounts that. The `cmd_build`
> file is `unlink`ed in `finally` but is never read or mounted: a pointless
> plaintext-credentials-on-disk exposure for zero effect. The port's `build`
> does **no** secrets marshalling; all secret handling (and its RAII cleanup)
> lives in `runner::run` (009).

So the port's `build` is: load `Config` (004) → apply the two `--sign-with-*`
overrides → pre-check `--log-file` → build a `RunOpts` (timeout, skip_build,
force, tls_verify, log sink = `--log-file` or stdout, no cancel token on the CLI
path) → `runner::run(desc_path, &config, opts)` (009). The CLI uses the
generated `gen_run_name()` container name (009). On `Err`, exit non-zero (see
Exit codes); on `Ok(report)`, the CLI does not need the report value (it is the
worker that consumes it natively, 011).

`build` runs in the operator's shell and **does not touch `HOME`** — the HOME
conditional is exclusively the `runner build` path's concern (below).

### `runner build` — in-container half, and the `HOME` hook (resolves 009's delegation)

009 delegates the conditional-`HOME` startup behaviour to 010. 010 specifies the
mechanism concretely: **as the first action of the `runner build` handler**
(before constructing the `Builder`), the binary replicates
`cbscore-entrypoint.sh:19-22`:

> If `HOME` is unset/empty **or** exactly `/`, set `HOME=/runner`. Otherwise
> leave it untouched (so an image that ships `HOME=/root` keeps it).

This is **scoped to the `runner build` subcommand only** — it is the sole path
that runs as PID 1 inside the builder image. `build`, `versions …`, and the root
group do not run it. After the `HOME` fix, `runner build` reads the descriptor
(`VersionDescriptor::read`, 002/006), constructs the `Builder` (007) with
`skip_build`/`force`/`tls_verify`, and runs it; a non-zero process exit on
failure is the only contract the host reads back (009 maps `rc != 0` →
`NonZeroExit`).

## Dropped surface (and how it relates to M1)

000's M1 enumerated only two **flag**-level drops (`--cbscore-path`,
`-e/--cbs-entrypoint`). The following **command**-level drops are later scope
decisions that **supersede** M1's flag enumeration; they are intentional, not
parity gaps:

- **`config init` and `config init-vault` (`config.py`) — dropped entirely.**
  Configuration is hand-authored (001: "configuration is hand-authored"); the
  interactive/flag-driven initializers and all their flags (`--components`,
  `--scratch`, `--containers-scratch`, `--ccache`, `--vault`, `--secrets`,
  `--for-systemd-install`, `--systemd-deployment`, `--for-containerized-run`,
  and `init-vault`'s `--vault`) are removed. The config/secrets/vault file
  formats are instead **documented in the cbscore-rs `README.md`** so an
  operator can author them by hand. This removes a large interactive surface
  (Click `confirm`/`prompt` flows) that has no place in a library-plus-thin-CLI
  design.
- **`advanced` group (`advanced.py`) — dropped.** It is an empty `hidden` group
  with no subcommands (`advanced.py:22-24`); there is nothing to port.
- **`--cbscore-path` and `-e/--cbs-entrypoint` on `build` — dropped** (the only
  M1 flag-level drops), because the binary-mount model (B1/009) makes a source
  path and an entrypoint script meaningless. Every other `build` flag is kept.

## Exit codes

Python exits operator commands with assorted `errno` values (`ENOTRECOVERABLE`,
`EINVAL`, `EEXIST`, `EACCES`, `ENOENT`) and `1`. The port does **not** reproduce
these numerically — nothing consumes the specific values:

- The **only** exit-code contract crossing the two-phase boundary is
  `runner build`'s **zero vs non-zero**, and 009 reads only `rc != 0` (not the
  value). So `runner build` exits `0` on success and non-zero on any failure.
- For operator commands (`build`, `versions …`) the port uses a small scheme:
  `0` success; `1` for a runtime failure (build error, S3/Vault/secrets error,
  descriptor write conflict); clap's conventional `2` for usage/parse errors.
  Errors are reported on stderr with context, as today.

## Fidelity notes

- **`--config` required+default → global default** — modelled as a defaulted,
  non-required global option; `versions create` ignores it; the file is read
  (and validated) only by the commands that load it.
- **`CBS_DEBUG=0` is off** — a shared Click-equivalent `BOOL` value parser
  (`1`/`true`/`t`/`yes`/`y`/`on` ⇒ on; their counterparts/empty/unset ⇒ off),
  never a presence check; superset of the entrypoint's `1`-only test; pinned by
  test (H3).
- **Binary-mount drops** — `--cbscore-path`, `-e/--cbs-entrypoint` gone (B1/M1);
  no `--debug`→env shim needed in the container (clap reads `CBS_DEBUG`).
- **`HOME` hook** — first action of `runner build` only; `HOME=/runner` iff
  unset/empty or `/` (`entrypoint.sh:19-22`); resolves 009's delegation.
- **Dead-secrets write fixed** — `build` does no secrets marshalling; `runner`
  owns it (009).
- **`--tls-verify` is value-taking** (`--tls-verify=<bool>`), not a bare flag —
  host parse and in-container emit (009) agree; parsed with the same shared
  Click-equivalent `BOOL` value parser as `CBS_DEBUG` (not clap's narrower
  `true`/`false`-only default), so Click-valid inputs (`yes`/`on`/`0`) are
  accepted.
- **`versions create VERSION` optional** (UUIDv7) — deliberate divergence (006).
- **`versions list` output** — approximate human text, no byte-parity (no
  machine consumer).
- **Command-level drops** (`config init`/`init-vault`, `advanced`) supersede
  M1's flag-level enumeration; manual config documented in the README.

## Testing

- **Parse parity**: each kept flag parses with its Python shape — `--skip-build`
  / `--force` as bare flags; `--tls-verify` value-taking and accepting the broad
  Click `BOOL` set (`false`/`no`/`off`/`0` and `true`/`yes`/`on`/`1`); `-c` /
  `--components-path` / `-o` repeatable and collected; `-c` required on
  `versions create`; `DESCRIPTOR` required existing on `build`.
- **`CBS_DEBUG`**: `CBS_DEBUG=1`→on, `CBS_DEBUG=0`→**off**, unset→off, `-d`→on.
- **`--config` default**: `versions create` runs with no config file present;
  `build`/`runner build`/`versions list` error clearly when the named config is
  missing.
- **In-container contract**: the `runner build` parser accepts exactly the argv
  009 emits
  (`--config … runner build --desc … --tls-verify=… [--skip-build] [--force]`).
- **`HOME` hook**: `runner build` sets `HOME=/runner` when unset/empty or `/`,
  leaves a non-`/` `HOME` (e.g. `/root`) untouched, and `build`/`versions …` do
  not alter `HOME`.
- **`build` glue**: `--sign-with-gpg-id`/`--sign-with-transit` fold into
  `config.signing` (creating it when absent); a pre-existing `--log-file` is
  rejected; no secrets temp file is created by `build` itself.
- **Dropped surface**: `config` and `advanced` are not registered commands;
  `--cbscore-path`/`-e` are not accepted by `build`.
