# 03 — Build Logs

## Overview

Tail, follow (live stream), and download build logs.

## Commands

### `cbc build logs tail <id> [options]`

Show the last N lines of a build log.

```
$ cbc build logs tail 42 -n 20

(showing 20 of 1,542 lines)
[2026-03-18T14:31:02] Building RPM packages...
[2026-03-18T14:31:03] Processing ceph-19.2.3...
...
```

**Endpoint:** `GET /api/builds/{id}/logs/tail?n={n}`

**Options:**

| Flag | Description | Default |
|---|---|---|
| `-n, --lines` | Number of lines | 30 |

Default matches the server's `default_tail_n()`.

**Response fields:**

```json
{
  "build_id": 42,
  "lines": ["line1", "line2", ...],
  "total_lines": 1542,
  "returned": 20
}
```

Display the count context before the lines:
`"(showing {returned} of {total_lines} lines)"`.

### `cbc build logs follow <id>`

Follow a build's log output in real-time via SSE.

```
$ cbc build logs follow 42

[2026-03-18T14:31:02] Building RPM packages...
[2026-03-18T14:31:03] Processing ceph-19.2.3...
...
(streams until build completes)

--- build 42 finished: success ---
```

**Endpoint:** `GET /api/builds/{id}/logs/follow`

**Transport:** Server-Sent Events (`text/event-stream`).

The server sends:


- `event: output` with `data: <line>` — a log line.
- `event: done` with `data: "build complete"` — stream
  ends.
- `id: <seq>` — sequence number for resume.

Note: the event name is `output` (not `log`).

**Implementation:** Use `reqwest-eventsource` crate:

```rust
use reqwest_eventsource::EventSource;

let mut es = EventSource::new(
    client.get_request(url)
)?;
let mut retries = 0;

while let Some(event) = es.next().await {
    match event {
        Ok(Event::Message(msg))
            if msg.event == "done" =>
        {
            break;
        }
        Ok(Event::Message(msg)) => {
            println!("{}", msg.data);
            retries = 0;
        }
        Ok(Event::Open) => { retries = 0; }
        Err(e) => {
            retries += 1;
            if retries >= 3 {
                eprintln!("stream error: {e}");
                break;
            }
        }
    }
}
```

`reqwest-eventsource` retries indefinitely per SSE
spec. The client counts consecutive reconnection errors
and closes after 3. Successful events reset the counter.

**After `done`:** The `done` event data is always the
literal string `"build complete"` — it does not carry
the build state. To show the final status, the client
makes a follow-up request: `GET /api/builds/{id}` and
prints the `state` field.

**Reconnection note:** Resumption via `Last-Event-ID`
is reliable while the build is active. For finished
builds, the server's seq-offset map may have been
cleared by GC — reconnection may replay from the start.

**Terminal state:** If the build is already finished
when `follow` starts, the server sends the full log
followed by `done` immediately.

### `cbc build logs get <id> [options]`

Download the full build log to a file.

```
$ cbc build logs get 42

log written to build-42.log (1.2 MB)
```

**Endpoint:** `GET /api/builds/{id}/logs`

**Options:**

| Flag | Description | Default |
|---|---|---|
| `-o, --output` | Output file path | `build-{id}.log` |

**Behavior:**

1. Send GET request with streaming response.
2. Write response body to file in chunks (8KB).
3. Print the output path and file size.

Uses `CbcClient::get_stream()` for streaming download
without buffering the entire response in memory.

## Error handling

- Build not found → 404 →
  `"build {id} not found"`.
- No log available (build still queued) → 404 or empty
  → `"no log available for build {id}"`.
- SSE connection lost → client retries up to 3 times
  (counted via consecutive errors). On exhaustion,
  print error and exit.
