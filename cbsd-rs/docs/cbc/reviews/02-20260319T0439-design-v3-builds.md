# Design Review: 02 — Build Operations (v3)

**Verdict: Approved.**

All v2 concerns resolved:

- `version` is positional for `build new`, `--version` for
  `periodic new`. Shared `BuildDescriptorArgs` excludes
  `version` — each command defines it separately.
- Nested struct construction documented: `dst_image` from
  `--image-name`/`--image-tag`, `build: BuildTarget` from
  `--distro`/`--os-version`/`--arch`.
- `--type` → `"version_type"` serialization explicitly noted.

No blockers. No major concerns.

## Minor Issues

- The `BuildRecord` field names used in `build list` and
  `build get` displays (`user_email`, `submitted_at`,
  `started_at`, `finished_at`) are still implied rather
  than named — but the design is clear enough for
  implementation.

## Strengths

- `version` positional vs `--version` flag split is clean
  and well-justified.
- Shared struct design is explicit about what is and isn't
  shared.
- Nested struct construction is documented.
- `SubmitBuildBody` wrapper, `versions` in components,
  `--all` semantics, and `--type` values are all correct.
