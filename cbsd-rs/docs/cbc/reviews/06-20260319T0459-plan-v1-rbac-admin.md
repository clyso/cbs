## Blocker

### B1 — `roles create` and `roles update` body uses `"capabilities"` but server expects `"caps"`

Plan Commit 1 shows the JSON body for `roles create` and
`roles update` as:

```json
{ "capabilities": ["<cap>", ...] }
```

The server's `CreateRoleBody` has field `caps: Vec<String>`
(`permissions.rs` line 73). The wire-format key is `"caps"`,
not `"capabilities"`. Sending `"capabilities"` will be
silently ignored by serde (unknown field), creating a role
with zero capabilities.

**Fix:** Change `"capabilities"` to `"caps"` in both the
`roles create` and `roles update` JSON examples.
