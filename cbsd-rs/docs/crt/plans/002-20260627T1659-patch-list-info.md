# CRT — Plan: `patch list` and `patch info`

> **Status:** Plan (**implemented** 2026-06-27). Implements design
> [`../design/002-20260627T1645-patch-list-info.md`](../design/002-20260627T1645-patch-list-info.md)
> (read-only `crt patch list` / `crt patch info`, `--json`, short-hash lookup).
> Seq `002`, shared with that design. Commit boundaries follow the `git-commits`
> skill (one capability per commit); two commits, ordered `list` → `info`.

## Scope

After this work an operator can see what `crt patch import` has stored: list
every imported patch (`crt patch list`) and inspect one by full hash or unique
short prefix (`crt patch info`), each with a `--json` mode for scripting. Both
commands are **read-only** — no blob, metadata, patch-id-index, draft, or
release is mutated. `crt patch delete` is **out of scope** (separate design).

The only new store surface is one enumerator, `list_patches`; everything else
reuses existing reads (`get_meta`, `get_patch_id`) and the `release list` /
`release info` CLI pattern.

## Commit 1 — `crt: list imported patches`

Delivers `crt patch list` (text + `--json`). Introduces the `list_patches`
enumerator with its first caller, so there is no dead-code intermediate.

**`crt-store/src/lib.rs`**

- Add to the `Store` trait:
  `async fn list_patches(&self) -> Result<Vec<Sha256>>;` (doc: "List the content
  addresses of all imported patches.").
- Add a `parse_meta_hash(loc: &StorePath) -> Option<Sha256>` helper: a meta
  object key is `patches/meta/sha256/<hex>.json` (4 parts); verify the first
  three segments, strip `.json` from the last, and `Sha256::try_from(hex)` (drop
  un-parseable entries with `.ok()`, mirroring `parse_release_key`).
- Implement `list_patches` on `ObjectBackedStore`: list the
  `patches/meta/sha256` prefix via `self.inner.list(...)` + `try_collect()`,
  then `filter_map(parse_meta_hash)` (same shape as the existing `list_keys`).
- **Store test:** `put_blob` + `put_meta` two patches → `list_patches` returns
  both hashes (assert set membership, order-agnostic); empty store → `[]`.

**`crt/src/patch.rs`** (new; `mod patch;` added to `main.rs`)

- `pub async fn list(store: &dyn Store) -> Result<Vec<PatchMeta>>`: call
  `list_patches`, `get_meta` each, collect, sort by the key
  `(subject, blob_hash.to_hex())` — `Sha256` has no `Ord`, so order the hash as
  its hex string. A failing `get_meta` (corrupt/unreadable) aborts the listing
  with that error (fail-loud), not a silent skip.
- `pub fn render_list(patches: &[PatchMeta]) -> String`: one
  `"{blob_hash}  {subject}\n"` line per patch.
- **Tests:** `list` sorts by `(subject, blob_hash.to_hex())` and is empty on an
  empty store; `--json` round-trips (serialize the `Vec<PatchMeta>` and parse it
  back to an equal value).

**`crt/src/main.rs`**

- Add `PatchCmd::List { #[arg(long)] json: bool }`.
- Dispatch arm (mirror `ReleaseCmd::List`): `open_store`,
  `let patches = patch::list(&store).await?;` then, if `json`,
  `println!("{}", serde_json::to_string_pretty(&patches)?)` and nothing else;
  else `print!("{}", patch::render_list(&patches))` +
  `eprintln!("{} patch(es)", patches.len())`.

**Commit message:**

```text
crt: list imported patches

Add `crt patch list` to enumerate the content-addressed patch store, with
`--json` for scripting. A new `list_patches` store method walks the
`patches/meta/sha256/` prefix; the CLI reads each patch's metadata and prints
`<blob_hash>  <subject>`, or a JSON array of the full records.
```

## Commit 2 — `crt: inspect an imported patch`

Delivers `crt patch info` (full hash or unique short prefix, text + `--json`).
Pure `crt`-side: consumes the existing `get_meta`/`get_patch_id` plus the
now-landed `list_patches` for prefix resolution; **no new `crt-store` change**.

**`crt/src/patch.rs`**

- `pub async fn find(store: &dyn Store, arg: &str) -> Result<PatchMeta>`:
  - **Validate first** with an explicit lowercase-hex predicate (`0-9a-f` only;
    `char::is_ascii_hexdigit` accepts uppercase, so it is not enough), then
    check the length is in `4..=64` — reject `<4` (too short) and `>64` (too
    long) with a clear error, before branching.
  - **Full (len 64):** `Sha256::try_from(arg)` + `get_meta`; map
    `is_not_found()` to `no patch with blob hash <hash>`.
  - **Prefix (len 4–63):** `list_patches`, keep hashes whose hex `starts_with`
    the arg — 0 → not-found error; 1 → `get_meta` it; >1 → ambiguous error
    listing the matches.
- `pub fn render_info(meta: &PatchMeta, equivalent: Option<&Sha256>) -> String`:
  the labeled block from the design (`blob_hash`, `patch_id`, `subject`,
  `author`, `authored`, `source_repo`, `provenance`, `cherry-picked-from` when
  non-empty, then the body); append the equivalence line when `equivalent` is
  `Some` and differs from `meta.blob_hash`. (The `by-patch-id` index stores only
  the first-seen blob, so the line is one-way — absent when inspecting that
  representative; this matches `import`.)
- **Tests:** full hash resolves; a unique prefix resolves to the same patch; an
  ambiguous prefix errors; a no-match prefix errors; a `<4`, `>64`, uppercase,
  or non-hex arg errors; `render_info` contains the key fields; the equivalence
  line appears when two blobs share a `patch_id`.

**`crt/src/main.rs`**

- Add `PatchCmd::Info { blob_hash: String, #[arg(long)] json: bool }`.
- Dispatch arm: `open_store`,
  `let meta = patch::find(&store, &blob_hash).await?;` then, if `json`,
  `println!("{}", serde_json::to_string_pretty(&meta)?)`; else look up the
  equivalence (`store.get_patch_id(&meta.patch_id).await?` filtered
  `!= meta.blob_hash`) and
  `print!("{}", patch::render_info(&meta, equiv.as_ref()))`.

**Commit message:**

```text
crt: inspect an imported patch

Add `crt patch info <blob_hash>` to show a patch's recorded metadata, accepting
a full hash or a unique short prefix (resolved via `list_patches`), with
`--json`. A read over the existing `get_meta`; no store change.
```

  `crt patch list --json | jq .` — a JSON array, no stderr noise.
- `crt patch info <full-hash>` and `crt patch info <short-prefix>` — same patch;
  `crt patch info <ambiguous-prefix>` — ambiguous error listing matches;

  `crt patch info <hash> --json | jq .` — the `PatchMeta` object.

## Invariants (do not regress)

- **Read-only.** No mutation path is added; blobs, metadata, the patch-id index,
  drafts, and releases are untouched.
- `crt-core` stays pure (no IO); new logic is the `crt-store` enumerator and the
  `crt` CLI module only.
- No new third-party dependencies (`object_store::list`, `serde_json`, `Sha256`
  are already in tree).
- stdout carries the result (text or a single JSON document); summaries and
  diagnostics go to stderr — matching the existing commands so the tool stays
  scriptable.
